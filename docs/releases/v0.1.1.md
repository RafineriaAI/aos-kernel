# AOS Kernel v0.1.1

Release tag: `v0.1.1`
Release commit: `c7182ed88186eff204c5ea7ef758810069e4a436`

## Why This Release Matters

`v0.1.1` is a publication-surface hardening release after the stable `v0.1.0`
public kernel tag. It keeps the kernel contract narrow while making the release
surface easier to audit.

The practical use case remains simple: a scanner, CI job, or evaluation step can
produce a bounded signal; AOS Kernel turns that signal into a deterministic
`PASS` / `WARN` / `BLOCK` handoff decision with replayable evidence. Reviewers
can later inspect the input digest, policy id, decision, and verification status
instead of reconstructing a one-off CI judgment from logs.

## Added

- GitHub Actions hardening by pinning third-party actions to full commit SHA
