# AOS Kernel v0.1.0

Release tag: `v0.1.0`
Release commit: `5f583bfb0b984bde317e2b09b782b2f60c944e13`

## Why This Release Matters

This is the first curated public AOS Kernel release surface. The practical use
case is simple: convert a bounded scanner, CI, or evaluation signal into a
repeatable `PASS` / `WARN` / `BLOCK` handoff decision with replayable evidence.

The release demonstrates the public kernel contract. It does not claim production
readiness, compliance, official signing, security of a target repository,
commercial ROI, or Python-Lean refinement correctness.

## Included

- deterministic reference kernel for bounded `PASS` / `WARN` / `BLOCK` decisions;
- minimal CLI demo path and Trusted Output v0 wrapper verification;
- tamper, fork, impersonation, rollback, and claim-boundary tests;
- public integrity manifest with SHA-256 digests;
- abstract Lean proof surface for selected public verdict-model properties;
- compact sanitized replay example.

## Boundaries

This public repository intentionally excludes private production policies,
customer workflows, signing keys, deployment hardening, commercial scoring,
private benchmark results, and enterprise delivery material.

Workflow products built on top of the kernel belong in separate product surfaces,
such as `RafineriaAI/aos-workflow-gate`.
