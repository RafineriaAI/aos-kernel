# Trusted Output v0

Trusted Output v0 is a small verification wrapper around an existing AOS JSONL
record. It does not change `PASS` / `WARN` / `BLOCK` semantics. It only makes the
published verdict easier to check for issuer metadata, record binding, version
floor and wrapper hash consistency.

```bash
aos trust emit --record examples/reports/public-replay-record.jsonl --output trusted-output.json
aos trust verify --input trusted-output.json --record examples/reports/public-replay-record.jsonl
```

Public demonstrator status:

```text
signature_status: UNSIGNED_NOT_OFFICIAL
official_aos_output: false
```

This is intentional. Public v0 output is structure-checkable and hash-checkable,
but it is not an official signed RafineriaAI/AOS verdict.

## Minimal Reading

| Field | Meaning |
| --- | --- |
| `verdict` / `decision` | The AOS decision copied from the source record. |
| `record_sha256` | Hash of the source AOS JSONL record. |
| `trusted_output_sha256` | Hash of the wrapper itself, excluding this field. |
| `manifest_sha256` | Hash of the public integrity manifest, or `missing` outside a source checkout. |
| `signature_status` | Public v0 status; normally `UNSIGNED_NOT_OFFICIAL`. |
| `official_aos_output` | `false` unless a future signed verifier returns official validity. |
| `claim_boundary` | Machine-readable reminder of what the public wrapper does not claim. |

## Verification Rule

A bare `PASS`, `WARN` or `BLOCK` is only a local verdict value. It becomes an
official RafineriaAI/AOS signed verdict only when a verifier returns
`OFFICIAL_VALID` against a trusted RafineriaAI issuer, trusted provenance and a
trusted root.

Public demonstrator outputs do not return `OFFICIAL_VALID`.

## Statuses

| Status | Meaning |
| --- | --- |
| `UNSIGNED_NOT_OFFICIAL` | Unsigned public output; not an official signed verdict. |
| `TAMPERED` | Wrapper hash, manifest hash, or source-record comparison failed. |
| `UNTRUSTED_FORK` | Non-RafineriaAI issuer or fork metadata. |
| `IMPERSONATION_ATTEMPT` | Output claims official RafineriaAI/AOS status without trusted verification. |
| `UNAUTHORIZED_BUILD` | Output claims official signature status, but public verification cannot validate it. |
| `ROLLBACK_DETECTED` | Output was generated by an AOS version below the declared trusted minimum. |

## Manifest Resolution

In a source checkout, Trusted Output uses `evidence/integrity_manifest.json` from
the repository. In package-only or private-runner contexts where that file is not
available, the wrapper records `manifest_sha256: missing` instead of reading an
unrelated manifest from the current working directory. A runner can provide an
explicit manifest with `AOS_INTEGRITY_MANIFEST=/path/to/integrity_manifest.json`.

A wrapper that records `manifest_sha256` should not be listed inside the same
manifest it references. The wrapper is checked by `trusted_output_sha256` and,
when provided, by comparison with the source JSONL record.

## Public Boundary

The public repository includes unsigned wrapper generation, verification of
structure and hashes, optional source-record comparison with `--record`, and
basic fork, impersonation and rollback detection inside the unsigned boundary.

It does not include signing keys, signer service, customer/license binding, TUF
trusted root, revocation infrastructure, commercial policies or official signed
reports.

Trusted Output v0 does not claim production readiness, legal validity, security
warranty, SLSA compliance, formal provenance, non-repudiation, cryptographic
authenticity or certification.
